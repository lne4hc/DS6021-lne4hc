# Load data
abalone <- read.csv(
  "https://archive.ics.uci.edu/ml/machine-learning-databases/abalone/abalone.data",
  header = FALSE,
  col.names = c(
    "sex", "length", "diameter", "height",
    "whole_wt", "shucked_wt",
    "viscera_wt", "shell_wt", "rings"
  )
)

abalone$sex <- factor(abalone$sex)

# 70/30 split
train_idx <- sample(1:nrow(abalone), size = 0.70 * nrow(abalone))

train <- abalone[train_idx, ]
test <- abalone[-train_idx, ]
  1. KNN

nn_model <- nnet(

cal_plot <- function(obs, pred, title) {
  data.frame(
    observed = obs,
    predicted = pred
  ) |>
    ggplot(aes(x = predicted, y = observed)) +
    geom_point(alpha = 0.35) +
    geom_smooth(method = "loess", se = FALSE) +
    geom_abline(slope = 1, intercept = 0, linetype = "dashed") +
    labs(
      title = title,
      x = "Predicted Rings",
      y = "Observed Rings"
    )
}

  1. The best model on the Abalone data was the neural network because it had the lowest test-set MSE, 4.243645. I am defining “best” as the model that makes the most accurate predictions on new test data. Since MSE measures the average squared difference between the predicted and observed number of rings, the lowest MSE means the model made the smallest prediction errors overall.

The neural network likely performed best because it can capture nonlinear patterns between the abalone measurements and the number of rings. However, “best” can depend on the goal. If the main goal is interpretability, linear regression with restricted cubic splines may be preferred because it had a similar MSE and is easier to explain. Random forest and ordinal regression also performed reasonably well, while KNN had the highest MSE in this run.

Overall, the model choice depends on whether the priority is prediction accuracy, calibration, interpretability, or simplicity. For this assignment, I would choose the neural network because I am using test-set MSE as my main definition of best.
